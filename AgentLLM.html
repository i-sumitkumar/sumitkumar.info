AutoCIRepair: Multi-Agent CI Build Repair

AutoCIRepair is a novel multi-agent system that automates the diagnosis and repair of Continuous Integration (CI) build failures. Built using LLM-powered agents, the system targets real-world repository-level CI workflows.

  • Uses GPT-4o-mini and LangChain to orchestrate three agents: Error Extractor, Debugger, and Developer
  • Analyzes raw logs, diffs, and call graphs to localize and fix CI failures
  • Validates patches using real-time GitHub Actions reruns
  • Tested on the LCA CI Build Repair Benchmark with 68 real-world failure cases
